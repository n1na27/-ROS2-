# 快速开始指南

## 第一步：准备参考图像

在 `Img` 文件夹中准备三张照片：

```
Img/
├── 1.jpg  ← 1号人的照片（清晰的正面照）
├── 2.jpg  ← 2号人的照片（清晰的正面照）
└── 3.jpg  ← 3号人的照片（清晰的正面照）
```

**图像要求：**
- 格式：JPG、JPEG 或 PNG
- 内容：一个人的清晰正面照
- 光线：充足且均匀
- 背景：简洁为佳

## 第二步：启动系统

打开命令行窗口，运行：

```bash
python run.py
```

系统会自动：
1. ✅ 检查 Python 版本
2. ✅ 检查依赖包
3. ✅ 检查摄像头
4. ✅ 加载参考图像
5. ✅ 提取人脸特征
6. ✅ 启动识别系统

## 第三步：测试识别

### 测试1号人
1. 让1号人站在摄像头前
2. 观察系统是否识别为"1"
3. 机器人右手应伸出食指
4. 左手保持不动

### 测试2号人
1. 让2号人站在摄像头前
2. 观察系统是否识别为"2"
3. 机器人右手应伸出食指+中指
4. 左手保持不动

### 测试3号人
1. 让3号人站在摄像头前
2. 观察系统是否识别为"3"
3. 机器人右手应伸出食指+中指+无名指
4. 左手保持不动

### 测试未识别
1. 让未录入的人站在摄像头前
2. 系统显示"Unknown"
3. 机器人双手自然下垂

## 键盘控制

运行时可用的快捷键：

- **Q** - 退出系统
- **R** - 重置触发状态（清除所有识别记录）
- **S** - 显示统计信息（识别次数、成功率等）

## 常见问题速查

### 问题：找不到参考图像
**解决：** 确保图像文件在 `Img` 文件夹中，且命名正确（1.jpg, 2.jpg, 3.jpg）

### 问题：识别不出来
**解决：** 
1. 检查参考图像质量
2. 调整 config.yaml 中的阈值
3. 确保光线条件良好

### 问题：左手也在动
**解决：** 
1. 确认代码已更新
2. 重启系统
3. 查看日志确认

### 问题：机器人没有动作
**解决：** 
1. 检查机器人服务是否启动
2. 查看错误日志
3. 确认 ROS2 环境正常

## 配置调整（可选）

如需调整识别灵敏度，编辑 `config.yaml`：

```yaml
frs:
  threshold_cosine: 0.60  # 识别阈值，范围0-1
                          # 值越大越严格
                          # 建议范围：0.55-0.70
```

调整建议：
- 识别太敏感（错误识别多）→ 提高阈值到 0.65-0.70
- 识别不够敏感（漏识别多）→ 降低阈值到 0.50-0.55
