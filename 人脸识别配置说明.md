# 人脸识别配置说明

## 参考图像准备

为了让系统能够识别不同的人并做出相应的手指动作，请按照以下步骤准备参考图像：

### 1. 图像命名规范

在 `Img` 文件夹中放置以下参考图像：
- `1.jpg` - 1号人的照片
- `2.jpg` - 2号人的照片
- `3.jpg` - 3号人的照片

**重要提示：** 文件名（不含扩展名）将作为身份标识，系统会自动识别为"1"、"2"、"3"。

### 2. 图像要求

- 格式：支持 `.jpg`、`.jpeg`、`.png`
- 内容：每张照片应包含一个清晰的正面人脸
- 光线：良好的光照条件，避免过暗或过亮
- 角度：尽量正面拍摄，避免侧脸或仰头
- 尺寸：无特殊要求，系统会自动处理

### 3. 识别逻辑

系统会根据识别到的人做出相应的手指动作：

| 识别身份 | 手指动作 | 说明 |
|---------|---------|------|
| 1号 | 右手伸出食指 | 左手保持初始状态 |
| 2号 | 右手伸出食指+中指 | 左手保持初始状态 |
| 3号 | 右手伸出食指+中指+无名指 | 左手保持初始状态 |
| 未识别 | 双手自然下垂 | 所有手指收拢 |

### 4. 系统启动

完成图像准备后，运行以下命令启动系统：

```bash
python run.py
```

系统启动后会：
1. 自动加载 `Img` 文件夹中的参考图像
2. 提取每张图像的人脸特征
3. 保存特征用于后续识别
4. 开始实时人脸识别

### 5. 测试验证

- 当1号人出现在摄像头前时，机器人右手应伸出食指
- 当2号人出现在摄像头前时，机器人右手应伸出食指和中指
- 当3号人出现在摄像头前时，机器人右手应伸出食指、中指和无名指
- 当未识别的人出现时，机器人双手应自然下垂

### 6. 调试说明

如果识别不准确，可以：
1. 检查 `config.yaml` 中的 `frs.threshold_cosine` 参数（识别阈值）
2. 确保参考图像清晰且人脸明显
3. 查看系统日志输出的识别置信度
4. 尝试更换更清晰的参考图像

### 7. 注意事项

- 左手始终保持初始状态不动
- 所有手指动作仅用右手完成
- 未识别时双手自然下垂，不做任何动作
- 系统有冷却时间机制，避免重复触发
